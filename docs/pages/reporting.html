[WIP] Reporting and Visualisation

SAE outputs need to be reported in a way that makes both the estimates and their uncertainty understandable. The aim is to help data users see where estimates are reliable, where they are model-driven, and where uncertainty remains high.

1 Reporting guidelines

Reporting should include enough information for users to interpret the estimates responsibly:

  • Estimate: the final direct or modelled value for each estimation area.
  • Uncertainty: confidence intervals, credible intervals, RMSE, or other method-specific uncertainty measures.
  • Evidence strength: sample size, effective sample size, or flags for no-sample and low-sample areas.
  • Model influence: whether the estimate is mostly direct-survey evidence or mostly model-driven.
  • Benchmarking: comparison with trusted higher-level totals or external benchmarks, where available.
Note

The uncertainty measure should be presented alongside the estimate, not as a separate technical appendix. Maps and charts should make uncertainty visible wherever the estimates themselves are shown.

2 Visualisation guidelines

Useful visual outputs include:

  • estimate maps;
  • uncertainty maps;
  • direct-versus-modelled estimate plots;
  • ranked line graphs with uncertainty intervals;
  • maps of modelled-minus-direct or modelled-minus-benchmark differences;
  • tables flagging areas with high uncertainty, low sample size, or strong model influence.

Good SAE visualisation should avoid giving a false sense of precision. A map of estimates alone can look authoritative even when many areas have wide intervals. Pairing estimate maps with uncertainty maps, or using line graphs with shaded intervals, helps keep the uncertainty visible.

3 Practical Tutorial

We will use the commute distance estimates exported from the Bayesian area-level workbook. The file contains the direct estimates, non-spatial FH estimates, spatial FH estimates, and their confidence or credible intervals.

Code 
library(dplyr)
library(readr)
library(stringr)
library(sf)

# Load boundaries for mapping
boundary_zip <- tempfile(fileext = ".zip")
boundary_url <- "https://raw.githubusercontent.com/shaunhoang/tutorial-site/main/data/tutorial-boundaries.zip"
download.file(boundary_url, boundary_zip, mode = "wb")
unzip(boundary_zip, exdir = tempdir())

msoa_boundaries <- st_read(
  file.path(tempdir(), "boundaries", "boundaries-msoa.geojson"),
  quiet = TRUE
) |>
  rename(domain_id = MSOA21CD) |>
  filter(str_starts(domain_id, "W"))

# Load exported estimates
estimate_url <- "https://raw.githubusercontent.com/shaunhoang/tutorial-site/main/data/bayesian_area_sae_commute_estimates.csv"

commute_estimates <- read_csv(
  estimate_url,
  show_col_types = FALSE
)

commute_map_data <- msoa_boundaries |>
  left_join(commute_estimates, by = "domain_id")
