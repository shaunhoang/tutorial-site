SAE Toolkit Lite

SAE Toolkit Lite is meant for quick adaptation of the materials to another survey, geography, or indicator. It assumes an area-level SAE workflow and a Bayesian modelling approach.

1 Minimum inputs and recommended schema

A SAE workflow usually needs:

  • a survey indicator file, with one row per sampled unit used in the estimate
  • a domain-size file, with one row per estimation area
  • a covariate file, when fitting an area-level model
  • boundary geometry to map results or derive spatial structure, if relevant
  • (optional) population unit file when fitting unit-level SAE

1.1 Survey indicator file:

A survey indicator file links each sampled unit to the chosen outcome, estimation area, and survey weight. Survey documentation should be kept alongside the file so that routing, missingness, design variables, and weight choice can be checked.

Column Meaning
unit_id Survey respondent, household, or sampled unit ID.
domain_id Estimation area code.
indicator Numeric outcome, or binary 0/1.
weight Survey weight selected for the estimand.
stratum (optional) survey stratum.

1.2 Domain-size file

If available, a domain-size file gives the denominator for each estimation area. The denominator must match the estimand’s target population rather than the full resident population by default.

Column Meaning
domain_id Estimation area code.
domain_size Denominator matching the target population.

1.3 Geometry file

Boundary geometry is useful for visualising estimates and for creating adjacency matrix for spatial models.

Column Meaning
domain_id Estimation area code, matching the domain-size file.
domain_name Human-readable area name.
geometry Polygon or multipolygon geometry for the estimation area.

1.4 Covariate file for area-level estimation

Covariates are area-level predictors available for every estimation area. They should be measured on the same geography as the direct estimates, or aggregated to that geography before modelling.

Column Meaning
domain_id Estimation area code.
covariate_* Area-level predictor available for every estimation area.

1.5 Files for unit-level estimation

Unit-level SAE needs two harmonised files:

  • The survey_unit_data file besides indicator also needs covariates used to fit the model
Column Meaning
unit_id Survey respondent, household, or sampled unit ID.
domain_id Estimation area code.
indicator Outcome to model.
weight Survey weight selected for the estimand.
stratum (optional) survey stratum.
covariate_* Individual or household covariate.
  • The population_unit_data file provides the prediction population (real or synthetic microdata), with one row per population unit, household, or population cell. The covariates must match those in survey_unit_data.
Column Meaning
unit_id Population person, household, or synthetic unit ID.
domain_id Estimation area code used to aggregate predictions.
covariate_* Matching individual or household covariate also used in survey_unit_data.
pop_weight Population count represented by the row (optional, in case population cells are used)

1.6 Recommended output schema

For diagnostics, each estimation script should end with an area-level results table using the same column names. This makes it easier to combine direct, area-level, spatial, and unit-level outputs.

Column Meaning
domain_id Estimation area code.
method Estimation method label.
estimate Point estimate for the estimation area.
se Standard error or posterior standard deviation, where available.
lower Lower confidence or credible interval bound.
upper Upper confidence or credible interval bound.

2 Downloadable scripts

Reusable scripts can be adapted once the input files follow the schema above.

Script Purpose
direct_estimates.R Calculate direct estimates by estimation area, sampling variances, and confidence intervals.
area_level_sae.R Fit Bayesian area-level model with covariates and spatial structure derived from boundary geometry, if relevant.
unit_level_sae.R Fit Bayesian unit-level model using survey microdata and population or synthetic-population covariates.
diagnostics.R Create comparison tables and plots for checking estimates, uncertainty, and model outputs.
